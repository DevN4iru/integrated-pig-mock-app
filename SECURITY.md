# Security Policy

Pigstep is proprietary software.

## Reporting security issues

Report security issues privately to the repository owner or authorized project maintainer.

Do not publish vulnerabilities, credentials, exploit details, database dumps, logs, backups, or private project information publicly.

## Secrets

Never commit:

- passwords;
- API keys;
- private keys;
- database dumps;
- production `.env` files;
- customer data;
- server credentials;
- backup archives.

## Access

Repository, server, database, and deployment access must be limited to authorized people only. Access should be removed when no longer needed.
